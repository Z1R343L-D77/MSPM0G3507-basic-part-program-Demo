# MSPM0G3507 基础功能演示程序

本仓库包含针对 MSPM0G3507 微控制器的一系列基础功能演示程序，每个功能模块都以 `.7z` 文件的形式提供。这些程序涵盖从基础工程模板到高级串口通信与传感器集成的多个功能，旨在帮助开发者快速上手和掌握 MSPM0G3507 的使用。

**重要提示：所有工程均基于 TI 官方 SDK 库开发，请确保正确配置 SDK 路径。**

## 模块功能描述

### 文件06\_ws\_00\_空文件工程模板

- **功能描述**: 提供一个空白工程模板，已正确配置官方 SDK 库路径。
- **目的**: 用于快速创建新项目的标准化起点。

### 文件06\_ws\_01\_定时器中断和时钟配置

- **功能描述**: 演示定时器中断和时钟的配置方法。
- **目的**: 帮助开发者掌握硬件定时器的基本用法。

### 文件06\_ws\_02\_按键

- **功能描述**: 实现按键输入的检测，包含消抖处理。
- **目的**: 学习按键消抖与输入处理的基本技巧。

### 文件06\_ws\_03\_普通串口接受与发送

- **功能描述**: 演示通过标准 UART 发送和接收数据。
- **目的**: 了解串口通信的基本原理和实现方法。

### 文件06\_ws\_04\_pwm控制电机

- **功能描述**: 使用 PWM（脉宽调制）来控制电机。
- **目的**: 学习嵌入式系统中的电机控制技术。

### 文件06\_ws\_05\_adc采集电压

- **功能描述**: 使用 ADC（模数转换）采集电压信号。
- **目的**: 掌握模拟信号采集与处理的基础。

### 文件06\_ws\_06\_编码器读取

- **功能描述**: 读取编码器数据以获取位置信息。
- **目的**: 实现与旋转或线性编码器的集成。

### 文件06\_ws\_07\_蜂鸣器报警

- **功能描述**: 实现蜂鸣器报警功能。
- **目的**: 用于声音提示或警报的实现。

### 文件06\_ws\_08\_pwm控制舵机

- **功能描述**: 使用 PWM 控制舵机角度。
- **目的**: 掌握舵机控制技术，用于位置调节。

### 文件06\_ws\_09\_修改时钟数为40Mhz

- **功能描述**: 修改系统时钟配置为 40MHz。
- **目的**: 学习系统时钟的修改及其影响。

### 文件06\_ws\_10\_读取mpu6050原始值

- **功能描述**: 读取 MPU6050 的原始数据（通过 I2C 接口）。
- **目的**: 初步接触 IMU（惯性测量单元）的基本用法。

### 文件06\_ws\_11\_读取6050\_roll\_pitch

- **功能描述**: 通过 MPU6050 数据计算 Roll 和 Pitch 角度。
- **目的**: 使用传感器数据实现姿态测量。

### 文件06\_ws\_12\_环形队列串口+超时中断

- **功能描述**: 结合环形队列和超时中断进行高效串口数据处理。
- **目的**: 演示高级串口处理方法，包括队列和超时管理。

## SDK 配置说明

### 官方 SDK 获取

本项目基于 TI MSPM0 SDK 开发，请从 TI 官网下载并安装：

- [TI MSPM0 SDK 下载页面](https://www.ti.com/tool/MSPM0-SDK)

### SDK 路径配置

解压工程后，请根据您的实际安装路径修改工程配置文件中的 SDK 路径。

**常见路径配置方式：**

**Keil MDK 项目**：

- 打开 Keil，加载工程
- Options for Target → C/C++ → Include Paths
- 添加或修改 SDK 头文件路径

  <br />

### 验证 SDK 配置

确保以下头文件能够被正确引用：

```c
#include "ti_msp_dl_config.h"
#include "ti/driverlib/dl_gpio.h"
#include "ti/driverlib/dl_uart.h"
// 其他所需头文件...
```

## 使用方法

1. 克隆本仓库：
   ```bash
   git clone https://github.com/D77go77/MSPM0G3507-basic-part-program-Demo.git
   ```
2. 解压 `.7z` 文件：
   - 请使用 [7-Zip](https://www.7-zip.org/) 或其他压缩工具解压各模块文件
   - 建议解压到不含中文字符的路径，避免编译问题
3. 配置 SDK 路径：
   - 根据您的 SDK 安装位置，修改工程中的 SDK 路径配置
   - 参考上方 [SDK 配置说明](#sdk-配置说明)
4. 编译和烧录：
   - 在您的 IDE 中打开工程
   - 编译工程，确保无错误
   - 使用调试器（如 XDS110）将程序烧录到 MSPM0G3507
5. 测试功能：
   - 按照每个模块中的代码和注释，运行并测试功能
   - 使用串口助手等工具查看调试输出（如有）

## 常见问题

### 编译错误：找不到头文件

- 检查 SDK 路径配置是否正确
- 确认 SDK 已正确安装
- 确保路径中不包含中文字符或特殊符号

### 烧录失败

- 检查调试器连接
- 确认开发板供电正常
- 检查 IDE 中的调试器配置

### 功能不工作

- 检查硬件连接是否正确
- 确认时钟配置（部分模块需要特定时钟频率）
- 查看代码注释中的引脚定义
