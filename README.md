# MSPM0G3507 基础功能演示程序

本仓库包含针对 MSPM0G3507 微控制器的一系列基础功能演示程序。这些程序涵盖从基础工程模板到高级串口通信与传感器集成的多个功能，旨在帮助开发者快速上手和掌握 MSPM0G3507 的使用。

---

## ⚠️ 重要提示：必须安装 MSPM0G3507 SDK 库

**所有工程均严格基于 TI 官方 MSPM0 SDK 库开发，没有 SDK 将无法编译和运行！**

- 本项目依赖 Texas Instruments (TI) 官方提供的 MSPM0 软件开发工具包 (SDK)
- SDK 包含了所有必要的驱动库、头文件和工具链支持
- 请务必先安装 SDK，再使用本项目的任何工程

## MSPM0 SDK 详细配置指南

### 1. 获取并安装 MSPM0 SDK

**必须步骤：**

1. 访问 TI 官方 MSPM0 SDK 下载页面：
   - [TI MSPM0 SDK 下载页面](https://www.ti.com/tool/MSPM0-SDK)

2. 下载适用于您开发环境的 SDK 版本（推荐最新稳定版）

3. 运行安装程序，按照向导完成安装
   - **建议安装路径**：`C:\ti\mspm0_sdk_x_xx_xx_xx`（不含中文字符）
   - 记住安装路径，后续需要在工程中配置

### 2. 验证 SDK 安装成功

安装完成后，请确保以下目录和文件存在：

```
C:\ti\mspm0_sdk_x_xx_xx_xx\
├── source\
│   └── ti\
│       └── driverlib\
│           ├── dl_gpio.h
│           ├── dl_uart.h
│           ├── dl_timer.h
│           └── ... (其他驱动头文件)
└── docs\
```

### 3. Keil MDK 工程 SDK 路径配置

**每个工程都需要配置 SDK 路径，请按以下步骤操作：**

1. 使用 Keil MDK 打开工程文件（`*.uvprojx`）
2. 点击菜单：**Project → Options for Target...** (或按 `Alt+F7`)
3. 切换到 **C/C++** 选项卡
4. 在 **Include Paths** 中添加 SDK 的头文件路径：
   - 例如：`C:\ti\mspm0_sdk_1_10_00_05\source`
5. 切换到 **Debug** 选项卡，确保调试器配置正确

### 4. 验证 SDK 配置

确保以下头文件能够被正确引用：

```c
#include "ti_msp_dl_config.h"
#include "ti/driverlib/dl_gpio.h"
#include "ti/driverlib/dl_uart.h"
#include "ti/driverlib/dl_timer.h"
// 其他所需头文件...
```

编译工程，如果没有报找不到头文件的错误，说明 SDK 配置成功。

## 使用方法

1. 克隆本仓库：
   ```bash
   git clone https://github.com/D77go77/MSPM0G3507-basic-part-program-Demo.git
   ```

2. **先安装 MSPM0 SDK**（非常重要！）
   - 参考上方 [MSPM0 SDK 详细配置指南](#mspm0-sdk-详细配置指南)
   - 确保 SDK 安装并配置完成

3. 打开工程：
   - 进入对应的模块目录
   - 使用 Keil MDK 打开 `keil\G3507.uvprojx`
   - 确认 SDK 路径配置正确

4. 编译和烧录：
   - 在 Keil 中编译工程，确保无错误
   - 使用调试器（如 J-Link、XDS110）将程序烧录到 MSPM0G3507

5. 测试功能：
   - 按照每个模块中的代码和注释，运行并测试功能
   - 使用串口助手等工具查看调试输出（如有）

## 常见问题

### 编译错误：找不到头文件

- 检查 SDK 路径配置是否正确
- 确认 SDK 已正确安装
- 确保路径中不包含中文字符或特殊符号

### 烧录失败

- 检查调试器连接
- 确认开发板供电正常
- 检查 IDE 中的调试器配置

### 功能不工作

- 检查硬件连接是否正确
- 确认时钟配置（部分模块需要特定时钟频率）
- 查看代码注释中的引脚定义
